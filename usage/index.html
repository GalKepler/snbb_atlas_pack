    Usage¶

    Fetching Pre-built Atlases¶

    The atlas files (NIfTI and GIFTI) are tracked by git-annex and can be fetched without running a build or installing Python dependencies.

    # Clone the dataset (metadata only — no large files yet)
datalad clone https://github.com/GalKepler/snbb-atlas-pack
cd snbb-atlas-pack

# Fetch a single atlas
datalad get atlases/atlas-TianS1/

# Fetch all Tian scales
datalad get atlases/atlas-TianS*/

# Fetch all Schaefer+TianS1 variants
datalad get atlases/atlas-Schaefer2018N*Tian2020S1/

# Fetch everything (large — check disk space first)
datalad get atlases/

    After fetching, atlas files are available as regular files at their BIDS paths. The TSV lookup tables are plain text tracked in git and are always present after cloning (no datalad get needed).

    Prerequisites (for building)¶

    • Python 3.13+
    • uv (fast Python package manager)
    • DataLad (for cloning and saving)
    • Access to the Tian MSA source files at /media/storage/yalab-dev/Tian2020MSA_v1.4/ (local path)

    Installation¶

    # Clone the DataLad dataset
datalad clone https://github.com/GalKepler/snbb-atlas-pack
cd snbb-atlas-pack

# Install Python dependencies
uv sync

    Building All Atlases¶

    uv run python main.py

    This runs the full build pipeline:

    1. Writes dataset_description.json
    2. Builds Tian S1–S4 (copies NIfTIs + generates TSVs)
    3. Builds HCPex (copies NIfTI + merges TSV from two source tables)
    4. Builds HCP-MMP (extracts GIFTI hemispheres from CIFTI + writes TSV)
    5. Builds 40 Schaefer2018+Tian combined atlases (downloads source data if needed, combines NIfTIs, concatenates TSVs)

    The build is idempotent — safe to re-run; existing files are overwritten.

    Schaefer2018 auto-download

    On first build, the 10 Schaefer2018 NIfTIs and LUT files (~2.5 GB) are automatically downloaded to sourcedata/Schaefer2018/. Subsequent runs skip the download.

    Saving with DataLad¶

    After building, save outputs with DataLad rather than plain git commit so that NIfTI/GIFTI files are properly annexed:

    datalad save -m "Rebuild all atlases"

    Python API¶

    After uv sync, the snbb_atlas_pack package is importable. It provides a high-level fetcher interface — no need to construct file paths manually.

    import snbb_atlas_pack as snbb

# List all 46 available atlas IDs
snbb.list_atlases()
# ['HCPMMP', 'HCPex', 'Schaefer2018N1000n7Tian2020S1', ..., 'TianS4']

# Fetch a volumetric atlas
atlas = snbb.get_atlas("TianS1")
atlas.maps       # Path → …/atlas-TianS1_space-MNI152NLin2009cAsym_res-01_dseg.nii.gz
atlas.maps_R     # None (volumetric atlases have no right-hemisphere path)
atlas.labels     # pd.DataFrame with columns: index, label, name, hemisphere, …
atlas.space      # 'MNI152NLin2009cAsym'
atlas.modality   # 'volumetric'

# Fetch a surface atlas (both hemispheres)
hcpmmp = snbb.get_atlas("HCPMMP")
hcpmmp.maps      # Path → …_hemi-L_dseg.label.gii
hcpmmp.maps_R    # Path → …_hemi-R_dseg.label.gii

# Single hemisphere
lh = snbb.get_atlas("HCPMMP", hemi="L")
lh.maps_R        # None

# Load the image with nibabel (maps is a Path, not a loaded object)
import nibabel as nib
img = nib.load(atlas.maps)
data = img.get_fdata()   # numpy array

# Get the prebuilt yabplot mesh directory
mesh_dir = snbb.get_mesh("TianS1", component="subcortical")

# Build meshes on demand (if derivatives/yabplot is not yet populated)
snbb.build_meshes("TianS1")   # single atlas
snbb.build_meshes()            # all atlases

    See the API Reference for full parameter and return-type documentation.

    Accessing Atlas Files (direct paths)¶

    Python (nibabel)¶

    import nibabel as nib
import pandas as pd

# Load a volumetric atlas
img = nib.load("atlases/atlas-TianS1/atlas-TianS1_space-MNI152NLin2009cAsym_res-01_dseg.nii.gz")
data = img.get_fdata()  # shape: (182, 218, 182)

# Load the region lookup table
tsv = pd.read_csv("atlases/atlas-TianS1/atlas-TianS1_dseg.tsv", sep="\t")
print(tsv.head())

    Python (nibabel — surface atlas)¶

    import nibabel as nib

# Load left hemisphere GIFTI
gifti = nib.load("atlases/atlas-HCPMMP/atlas-HCPMMP_space-fsLR_hemi-L_dseg.label.gii")
labels = gifti.darrays[0].data  # shape: (32492,) — one label per vertex

    Generating Atlas Visualizations¶

    Atlas figures are generated by scripts/visualize_atlases.py using yabplot. Figures are saved to atlases/<atlas-name>/figures/ and intermediate mesh/vertex data is cached in derivatives/yabplot/ for fast re-plotting.

    uv run python scripts/visualize_atlases.py

    The script is idempotent — it skips mesh building if the cache already exists. Running it a second time only re-renders the PNGs.

    Output files per atlas family¶

    Atlas family figures/ output
    atlas-TianS{N} atlas-TianS{N}_subcortical.png
    atlas-HCPex atlas-HCPex_cortical.png, atlas-HCPex_subcortical.png
    atlas-HCPMMP atlas-HCPMMP_cortical.png
    atlas-Schaefer2018N{N}n7Tian2020S{S} …_cortical.png, …_subcortical.png

    Cached derivatives¶

    Path Contents
    derivatives/yabplot/atlas-TianS{N}/ VTK subcortical meshes (reused by all Schaefer+TianS{N})
    derivatives/yabplot/atlas-HCPex/ Cortical surface CSV + wb label file; subcortical/ VTK meshes
    derivatives/yabplot/atlas-HCPMMP/ HCPMMP.csv vertex array, HCPMMP.txt LUT
    derivatives/yabplot/atlas-Schaefer2018N{N}n7/ Cortical surface CSV + wb label file

    Using Prebuilt yabplot Meshes¶

    Once derivatives/yabplot/ is populated, you can create custom visualizations — with your own per-region data overlaid — without rebuilding meshes. This is the recommended workflow for downstream analysis scripts.

    Subcortical plot (Tian, HCPex subcortex)¶

    import yabplot as yab
import numpy as np

# Load region metadata to know index ordering
import pandas as pd
tsv = pd.read_csv("atlases/atlas-TianS1/atlas-TianS1_dseg.tsv", sep="\t")

# Your per-region values (e.g. FA, CBF, connectivity strength)
# Must have length == number of regions; order matches TSV index column
data = np.random.rand(len(tsv))

yab.plot_subcortical(
    data=data,
    custom_atlas_path="derivatives/yabplot/atlas-TianS1",
    export_path="my_subcortical_figure.png",
    display_type="none",
    figsize=(2000, 700),
)

    Cortical plot (HCPex, HCPMMP, Schaefer+Tian)¶

    import yabplot as yab

yab.plot_cortical(
    data=None,                                  # None = plain parcellation colours
    custom_atlas_path="derivatives/yabplot/atlas-HCPex",
    export_path="my_cortical_figure.png",
    display_type="none",
    figsize=(2000, 800),
)

    Schaefer+Tian subcortical — reuse Tian meshes¶

    All 40 Schaefer+Tian combinations share subcortical meshes with the corresponding Tian atlas. There is no need to rebuild subcortical meshes per Schaefer resolution:

    # S=2 subcortical mesh works for any Schaefer+TianS2 variant
yab.plot_subcortical(
    data=None,
    custom_atlas_path="derivatives/yabplot/atlas-TianS2",   # shared mesh
    export_path="schaefer400_tians2_subcortical.png",
    display_type="none",
)

    Getting QSIRecon / PennLINC Atlases¶

    The following atlases are already distributed by QSIRecon's AtlasPack and are not included here to avoid duplication:

    Atlas Regions
    4S series (4S156–4S1056) 156–1056
    Schaefer 2018 (7-network, 17-network) 100–1000
    AAL 116
    AICHA384Ext 384
    Brainnetome246Ext 246
    Gordon333Ext 333

    Option 1 — DataLad clone (recommended): 

    datalad clone https://github.com/PennLINC/AtlasPack
cd AtlasPack
datalad get atlas-4S456Parcels/   # download a specific atlas

    Option 2 — TemplateFlow Python client: 

    import templateflow.api as tflow
files = tflow.get("MNI152NLin2009cAsym", atlas="Schaefer2018", resolution=1)

    Option 3 — Pre-bundled in QSIRecon containers: Atlases are available at /opt/templateflow/ inside QSIRecon Docker/Singularity images.
